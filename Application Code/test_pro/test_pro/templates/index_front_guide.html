Coding Conventions

Coding conventions are a set of guidelines for a specific programming language that recommend programming style, practices, and methods for each aspect of a program written in that language.

  • 100% Compatible with twitter bootstrap
  • Valid HTML5 code and well structured
  • Really updatable and easy to customize

DECLARE A DOCTYPE

The DOCTYPE declaration should be in the first line of HTML. Actually, it activates the standard mode in all browser. It is recommended that you use the HTML doctype.

CLOSE YOUR TAG

Leaving some tags open is simply a bad practice. Only self-closing tags br,hr,img,link,meta,inputetc are valid. Normal elements can never have self-closing tags.

USE A CSS RESET

CSS reset will ensure that all elements have no particular style so you can define your own.

AVOID UNITS ON ZERO VALUES

One way to easily cut down on the amount of CSS we write is to remove the unit from any zero value. A zero will always be a zero.

USE PRACTICAL ID AND CLASSES NAMES AND VALUES

We should only give elements an ID attribute if they are unique. Classes can be applied to multiple elements that share the same style properties. It is always preferable to name something ID or class, by the nature of what it is rather than by what it looks like.

WRITE CSS USING MULTIPLE LINES AND SPACES

It is important to place each selector and declaration on a new line.

CONSIDER PLACING JAVASCRIPT AT THE BOTTOM

When loading a script, the browser cannot continue until the entire file has been loaded. If we have JavaScript files in order to add functionality, we should place those files at the bottom, just before the closing body tag.

MODULARIZE STYLES FOR REUSE

CSS is built to allow styles to be reused, specifically with the use of classes. For this reason, styles assigned to a class should be modular and available to share across elements as necessary.

CHECKING IN CROSS-BROWSER WHILE DEVELOPING

Cross-browser issues are a major problem for front-end developers, especially due to IE. If we test our documents on Firefox/IE/Chrome while writing it, cross-browser rendering problems will be much easier to fix.
